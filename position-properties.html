Position

The position property controls the type of positioning used by an element within its parent elements.

The effect of the position attribute depends on a lot of factors, for example the value of position property of parent elements.

In this tutorial, we are going to look at how positioning will change the relationship between parent and child elements.

Parent

A parent element is an element that contains other elements.

Child

All elements that are contained within that parent are called "child" elements.

Properties

position: static;

Default. Object has no special positioning; it follows the layout rules of HTML. Values of top, bottom, left and right have no impact.

position: absolute;

Object is positioned relative to parent element's position—or to the body object if its parent element is not positioned—using the top and left properties.

position: relative;

Object is positioned according to the normal flow, and then offset by the top and left properties.

position: fixed;

An element with fixed position is positioned relative to the browser window. It will not move even if the window is scrolled. See any :)

What is z-index?

The z-index property specifies the stack order of an element. An element with greater stack order is always in front of an element with a lower stack order.

parent: z-index: 100
child: z-index: 1

position: fixed;

A fixed position element is positioned relative to the viewport, or the browser window itself. The viewport doesn't change when the window is scrolled, so a fixed positioned element always stays in place as the page is scrolled.

Tips:

  • Can be used on navigations to keep them visible at all times.
  • Used more rarely compared to other position properties.
  • Fixed elements are removed from the normal flow.
  • Can overlap other elements.
