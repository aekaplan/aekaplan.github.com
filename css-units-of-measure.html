CSS Units of measure

With so many units of measurement available, which unit is the best for the job? Let's find out!

  • px
  • em
  • rem
  • %

px

Pixels are fixed-size units. Don't let the name fool you, pixels don't really have anything to do with screen pixels.

It's actually considered an angular measurement. I started using pixels but have since moved to ems and percents. More on that later.

Good

  • For setting a max-width on containers and media queries
  • Fine-grained control
  • Great for any time you want to define a particular size

Bad

  • Inflexible - 10px is always 10px
  • Doesn't scale according to the width of the browser
  • Better ways to size text for responsive web design using percents em or rems

em

An em it is a unit that is relative to the currently chosen font size. This unit is highly recommended for responsive web design.

Good

  • Flexible
  • Can reserve an amount of space relative to the current font-size
  • Easily change your font size in one place
  • Keeps relationships with margin and padding using ems

Bad

  • Nested lists will cascade and grow the font size of the child lists
  • Numbers can be confusing
  • This is where rems come in
